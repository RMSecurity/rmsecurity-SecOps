# Peer Review Process
### rmsecurity | 02-QUALITY

## What Peer Review Is

Peer review is a deliberate, structured read of a deliverable by someone
other than its author, using the appropriate quality checklist, before
the deliverable proceeds to the next gate.

In a solo operation, peer review means stepping away from the work for
at minimum 2 hours, returning with fresh eyes, and reviewing as if you
did not write it. It is not skipped because there is only one analyst.
It is performed with a time gap to simulate a different perspective.

As the team grows, peer reviews are performed by a second person.

---

## When Peer Review is Required

| Deliverable | When | Checklist |
|------------|------|----------|
| Each finding | After writing, before adding to report | Gate 2 |
| Technical report draft | Before Gate 3 | Gate 3 |
| Executive summary draft | Before Gate 3 | Checklist: executive report |
| Evidence package | During testing, at collection | Checklist: evidence |
| Final delivery package | Before Gate 4 | Gate 4 |

---

## Peer Review Rules

### For the reviewer

- Read every word — no skimming
- Use the checklist — do not rely on memory
- Mark every issue, no matter how minor
- Write specific comments — "section 3, finding FND-003, the impact description does not mention business risk"
- Do not fix issues yourself — document them for the author to fix
- Complete the peer review form for every review performed

### For the author

- Do not be present during the review — do not explain or defend as the reviewer reads
- Address every issue raised — do not selectively fix
- If you disagree with a comment, document the disagreement and the rationale
- After corrections, request a second pass if the reviewer marked critical issues

### Time requirements

| Document type | Minimum review time |
|-------------|-------------------|
| Individual finding | 15 minutes |
| Full technical report | 2–4 hours |
| Executive summary | 1 hour |
| Complete delivery package | 30 minutes |

---

## Solo Operator Protocol

When operating alone:

1. Complete the draft
2. Step away for minimum 2 hours (ideally overnight for reports)
3. Return and review using the checklist as if reading for the first time
4. Complete the peer review form — even solo, even for yourself
5. Record the review in the engagement folder

The discipline of writing down the review creates accountability
even without a second person. It also builds the habit so that
when a second consultant joins, the process is already natural.

---

## Escalation

If a reviewer finds an issue that cannot be resolved by the author
(disagreement on severity, contradictory evidence, scope question):

1. Document the disagreement in the peer review form
2. Both parties write their position
3. If unresolved, the more conservative position wins
   (lower severity = better for the client but safer for rmsecurity's credibility)
4. Log the disagreement as a potential lesson learned
